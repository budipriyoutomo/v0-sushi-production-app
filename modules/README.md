# Modules

This folder is reserved for future domain modules. Existing screens remain in `app/` and `components/` to avoid a disruptive rewrite.

Recommended future module shape:

- `modules/kitchen/`
- `modules/production/`
- `modules/operation/`
- `modules/admin/`
- `modules/reports/`

Move code here only when a feature is being actively changed or when shared domain logic is being extracted.
